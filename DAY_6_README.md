Day 6 - Functional Testing & End-to-End Testing (SauceDemo)

Objective
To understand Functional Testing concepts and perform End-to-End testing by creating test scenarios,
test cases, regression suite, and test execution results using the SauceDemo application.

 Application Under Test (AUT)
Website: SauceDemo  
URL:https://www.saucedemo.com/

Test Credentials:
- Username: `standard_user`
- Password: `secret_sauce`

Functional Testing Approach
Functional testing was performed to validate application features based on expected requirements. Testing was done module-wise to verify input, process, and output behavior.

Modules Covered:
- Login / Logout
- Product listing page
- Sorting (Price and Name sorting)
- Add to Cart / Remove from Cart
- Cart page validation
- Checkout process
- Order confirmation flow

Testing Types Covered
- Smoke Testing
- Sanity Testing
- Regression Testing
- End-to-End (E2E) Testing

 Deliverables Prepared
- Smoke Test Checklist
- Sanity Test Cases
- Regression Test Suite
- 35 End-to-End Test Scenarios (Happy Path, Negative, Alternative, Edge cases)
- Test Execution Results

End-to-End Flow Tested
Login → Products Page → Add to Cart → Cart → Checkout → Enter User Info → Overview → Finish → Confirmation → Logout

Conclusion
Functional testing ensured each feature worked as expected, while E2E testing validated complete user workflows.
Regression suite was created to quickly verify critical features after changes.
