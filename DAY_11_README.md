Day 11 - Introduction to Test Automation

Objective
To understand the basics of automation testing, learn when automation is useful, explore automation tools and frameworks, set up automation environment, and execute the first Selenium automation script.

Tools Used
- Java (JDK)
- Eclipse IDE
- Selenium WebDriver
- Maven (Dependency Management)
- Chrome Browser / ChromeDriver

Topics Covered
- What is Test Automation
- Benefits of Automation
- Challenges in Automation
- ROI (Return on Investment) of Automation
- When to Automate vs When Not to Automate
- Automation Testing Pyramid
- Automation Tools Overview
- Automation Framework Types

When to Automate
✅ Regression testing (repetitive tests)  
✅ Stable features  
✅ Smoke testing  
✅ Data-driven testing  

When NOT to Automate
❌ Exploratory testing  
❌ Frequently changing UI  
❌ One-time test cases  
❌ Usability testing  


Automation Tools Overview
- **Selenium** – Web automation  
- **Appium** – Mobile automation  
- **Cypress** – Modern web testing  
- **Playwright** – Cross-browser automation  
- **JUnit / TestNG** – Test execution frameworks  
- **Postman / REST Assured** – API testing automation  

Automation Frameworks
- Data-Driven Framework  
- Keyword-Driven Framework  
- Hybrid Framework  
- Page Object Model (POM)  

Tasks Completed
- Setup automation environment (Java + Eclipse + Maven + Selenium)
- Created and executed first Selenium script (demo)
- Prepared automation tools comparison
- Created when-to-automate checklist
- Prepared automation strategy document

Deliverables Completed
- Automation Environment Setup
- First Selenium Script (Demo)
- Automation Tools Overview
- When to Automate Checklist
- Automation Strategy Document
- README with Automation Basics

 Conclusion
Day 11 provided a clear understanding of automation testing fundamentals and hands-on experience in executing a basic Selenium automation script using Java.
